Flutter横跨Android、iOS、MacOS、Windows、Linux等多个系统，还可以打包成Web程序运行在浏览器上。

1 概念

  • 概念
    • Flutter的核心概念：组件、构建、状态、框架等。
    • 组件(Widget)
      • 指Flutter应用程序用户界面的基本构建块，定义如下。
        • 一种动画处理，例如缓动。
        • 一种布局，例如填充或滚动。
        • 一个文本样式，例如字体或颜色。
        • 一个手势处理，GestureDetector。
        • 一个界面组件，例如按钮或输入框。
      • 通过属性改变组件的状态以及回调方法的处理，通过方法提供了组件的一些功能扩展。
      • 组件嵌套的组合方式带来的最大好处即解耦，最基础的组件类是Widget，其他皆继承。
        • 无状态组件(StatelessWidget)，界面不发生变化，例如Text、AssetImage等。
        • 有状态组件(StatefulWidget)，界面会发生变化，例如Scrollable、Animatable等。
    • 重写Widget的build构建方法：@protected Widget build(BuildContext context);。
      • 构建组件即创建一个Widget，返回值也是一个Widget对象，都属于Widget的实例。
      • BuildContext即构建Widget上下文获取数据，每一个BuildContext对应一个Widget。
        • Theme.of(context)：获取主题。
        • Navigator.push(context, route)：压栈。
    • 状态(State)是一个组件的UI数据模型，是组件渲染时的数据依据。
      • Flutter程序的运行可看成一个巨大的状态机，用户的操作、请求API和系统事件的触发都是触发点。
      • 触发点通过调用setState方法推动状态机进行响应，实现不同的状态渲染UI，让UI与数据保持一致。

2 环境搭建

  • 环境搭建
    • 虚拟机下Win10企业版(Win10-1909-Business-Editions-DVD-x64)、Chrome。
    • 下载并安装Flutter依赖的命令行工具Git for Windows：Git-2.37.1-64-bit.exe。
    • Flutter SDK：Flutter-Windows-3.0.5-Stable.zip。
      • 解压到指定位置，找到文件夹中的flutter_console.bat文件，双击运行。
      • 注意不要将Flutter SDK安装到高权限路径下，例如C盘的Program Files。
    • 添加环境变量：搜索控制面板—>用户账户—>更改我的环境变量—>环境变量。
      • 新建变量名PUB_HOSTED_URL：变量值https://pub.flutter-io.cn/。
      • 新建变量名FLUTTER_STORAGE_BASE_URL：变量值https://storage.flutter-io.cn/。
      • 变量Path中添加：变量值C:\Program\Flutter\bin(即Flutter SDK安装路径的bin目录)。
    • 编辑器安装与设置
      • 安装Visual Studio：VisualStudioSetup.exe，使用C++的桌面工具。
        • 勾选用于Windows的C++ CMake工具。
        • 勾选Windows 10 SDK(10.0.19041.0)。
        • 勾选MSVC v143-VS 2022 C++ x64/86生成工具(最新)。
      • 安装Android Studio：Android-Studio-2021.2.1.15-Windows.exe，需要梯子。
      • 添加环境变量ANDROID_SDK_HOME：变量值C:\Program\Avd(模拟器存放的位置)。
      • 设置Android模拟器：虚拟机貌似无法启动模拟器，建议将环境搭建在主机上进行。
        • 启动Android Studio—>More Actions—>Virtual Device Manager—>Create device。
        • 选设备Phone·Pixel—>Next—>选镜像R—>Download—>Next—>验证配置信息—>Finish。
      • 安装插件：Android Studio启动界面—>Plugins—>Marketplace—>搜索Flutter和Dart—>Install。
        • Flutter插件：支持Flutter开发工作流(运行、调试、热重载等)。
        • Dart插件：提供代码分析(输入代码时进行验证、代码补全等)。
    • 文件夹下鼠标右键选择Git Bash Here，命令窗口模式输入flutter doctor进行查看。
      • cmdline-tools component is missing：Android Studio缺少cmdline-tools工具。
        • 启动Android Studio—>More Actions—>SDK Manager—>SDK Tools。
        • 勾选Android SDK Command-line Tools (latest)—>Apply—>Finish—>OK。
      • Android license status unknown.：需要添加Android license。
        • 搜索cmd，以管理员身份运行，命令窗口下执行flutter doctor --android-licenses。
        • 在提示是否接受许可时，输入y进行回车确认，最后重新输入flutter doctor进行查看。

3 Flutter基础

  • Flutter基础
    • 启动Android Studio—>New Flutter Project—>Flutter—>…(Flutter SDK的安装路径)—>OK—>Next。
    • 将Project Name命名为my_first_app—>Finish—>等待几分钟，自动创建工程—>No Device Selected。
    • Open Android Emulator: Pixel API 30(启动之前设置的Android模拟器)—>cmd下执行flutter doctor。
      • ! Device 127.0.0.1:5554 is offline.
      • ! Device emulator-5554 is not authorized.
      • You might need to check your device for an authorization dialog.
      • 说明Android Studio未正常连接到模拟器设备，点击Android Studio任务栏右侧的Device Manager。
      • Create device—>Phone·Pixel—>Next—>选镜像x86 Images的R(Google APIs而非Google Play)。
      • Download—>Next—>验证配置信息—>Finish—>重新选择Open Android Emulator: Pixel API 30。
    • 点击Run'main.dart'时，底部显示Running Gradle task 'assembleDebug'...。
      • 这是因为Gradle的Maven仓库在国外，需使用国内例如阿里云的镜像地址进行替换，或借助梯子访问。
      • 可通过修改flutter.gradle、build.gradle和resolve_dependencies.gradle文件内容替换地址，具体操作。
    • 点击Debug按钮，启动官方示例程序，在模拟器中点击+按钮可自动加1(基于Material Design的应用程序)。
    • 清空项目中main.dart文件的内容，输入以下代码，重新运行此程序，这样第一个Flutter程序就运行出来了。
import 'package:flutter/material.dart';

void main() => runApp(MyApp());

class MyApp extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      title: 'Welcome to Flutter',
      home: Scaffold(
        appBar: AppBar(
          title: Text('Welcome to Flutter'),
        ),
        body: Center(
          child: Text('Hello World!'),
        ),
      ),
    );
  }
}

3-1 入口程序

  • 入口程序
    • 每个Flutter项目的lib目录里都会有一个main.dart文件，该文件中有一个main()函数。
    • Flutter使用Dart语言开发，main()函数是程序入口，程序运行时即第一个执行的函数。
    • main()函数中只调用了runApp()函数，使用该函数可以将给定的根组件填满整个屏幕。
      • Flutter是Dart语言的移动应用框架，runApp()函数则是Flutter框架的入口函数。
      • 不调用runApp()函数时执行的就是一个Dart控制台应用，屏幕上将不显示内容。
// void main() => runApp(Widget app);
// Dart语言特有的速写形式，展开代码如下
void main() {
  return runApp(Widget app);
}

3-2 Material Design

  • Material Design
    • dart文件首行是导包flutter/material.dart，是实现Material Design设计风格的基础包。
    • 其中包含了文本输入框Text、图标Icon、图片Image、形排列布局Row、列排列布局Column。
    • 装饰器Decoration、动画等组件，可将其理解为网页中常见的按钮、标题或选项框等组件库。
    • Material Design是谷歌推出的一套视觉设计语言，而Flutter采用的就是Material Design风格。
import 'package:flutter/material.dart';

4 Flutter主题

  • Flutter主题
    • 为在整个应用中使用同一套颜色和字体样式，可以使用主题这一方式。
    • 可以使用全局主题，或使用Theme定义应用程序局部的颜色和字体样式。
    • Material Widgets将使用主题为AppBars、Buttons、Checkboxes等设置背景颜色和字体样式。

4-1 创建主题

  • 创建主题
    • 将ThemeData提供给MaterialApp构造函数，在整个应用程序中共享包含颜色和字体样式的主题。
    • 如果没有提供主题，Flutter就会自动创建一个默认主题，ThemeData的主要属性和描述如下所示。
      属性名 类型 说明
      accentColor Color 前景色(文本、按钮等)
      accentColorBrightness Brightness accentColor的亮度，用于确定突出颜色的文本和图标的颜色(例如FloatingButton上的图标)
      accentIconTheme IconThemeData 与突出颜色对比的图标主题
      accentTextTheme TextTheme 与突出颜色对比的文本主题
      backgroundColor Color 与primaryColor对比的颜色(例如用作进度条的剩余部分)
      bottomAppBarColor Color BottomAppBar的默认颜色
      brightness Brightness 应用程序整体主题的亮度，由按钮等Widget使用，以确定在不使用主色或强调色时要选择的颜色
      buttonColor Color Material中RaisedButtons使用的默认填充色
      buttonTheme ButtonThemeData 按钮等控件的默认配置，例如RaisedButton和FlatButton
      canvasColor Color MaterialType.canvas Material的默认颜色
      cardColor Color Card的颜色
      chipTheme ChipThemeData 用于渲染Chip的颜色和样式
      dialogBackgroundColor Color Dialog元素的背景色
      disabledColor Color 用于Widget无效的颜色，包括任何状态，例如禁用复选框
      dividerColor Color Dividers和PopupMenuDividers的颜色，也用于List-Tiles中间和DataTables的每行中间
      errorColor Color 用于输入验证错误的颜色，例如在TextField中
      hashCode int 对象的哈希值
      highlightColor Color 用于动画或指示菜单被选中的高亮颜色
      iconTheme IconThemeData 图标主题，图标的默认配置
      indicatorColor Color 选项选中的指示器颜色
      inputDecorationTheme InputDecorationTheme InputDecorator、TextField和TextFormField的默认InputDecoration值基于此主题
      platform TargetPlatform Widget需要适配的目标类型
      primaryColor Color App主要部分的背景色(ToolBar、Tabbar等)
      primaryColorBrightness Brightness primaryColor的亮度
      primaryColorDark Color primaryColor的较暗版本
      primaryColorLight Color primaryColor的较亮版本
      primaryIconTheme IconThemeData 与主色对比的图标主题
      primaryTextTheme TextThemeData 与主色对比的文本主题
      scaffoldBackgroundColor Color 作为Scaffold基础的Material默认颜色，典型Material应用或应用内页面的背景颜色
      secondaryHeaderColor Color 有选定行时PaginatedDataTable标题的颜色
      selectedRowColor Color 选中行时的高亮颜色
      sliderTheme SliderThemeData 用于渲染Slider的颜色和形状
      splashColor Color 动画的颜色
      splashFactory InteractiveInkFeatureFactory 定义InkWall和InkResponse生成的动画外观
      textSelectionColor Color 文本字段中选中文本的颜色，例如TextField
      textSelectionHandleColor Color 用于调整当前文本中指定部分的颜色
      textTheme TextTheme 与卡片和画布对比的文本颜色
      toggleableActiveColor Color 用于突出显示切换Widget(例如Switch、Radio和Checkbox)活动状态的颜色
      unselectedWidgetColor Color 用于Widget处于非活动但已启用状态的颜色，例如未选中的复选框，通常与accentColor形成对比
      runtimeType Type 表示对象的运行时类型 
// 默认主题示例
MaterialApp(
  title: title,
  theme: ThemeData(
    brightness: Brightness.dark,
    primaryColor: Colors.lightBlue[800],
    accentColor: Colors.cyan[600],
  ),
);

4-2 局部主题

  • 局部主题
    • 要在应用程序的某一部分使用特殊的颜色，需覆盖全局主题。
    • 通过创建特有的主题数据，或通过扩展父主题的方法来实现。
// 实例化一个ThemeData并将其传递给Theme对象
Theme(
  // 创建一个特有的主题数据
  data: ThemeData(
    accentColor: Colors.yellow,
  ),
  child: FloatingActionButton(
    onPressed: () {},
    child: Icon(Icons.add),
  ),
);

// 扩展父主题时无须覆盖所有的主题属性，可以使用copyWith方法来实现
Theme(
  // 覆盖accentColor为Colors.yellow
  data: Theme.of(context).copyWith(accentColor: Colors.yellow),
  child: FloatingActionButton(
    onPressed: null,
    child: Icon(Icons.add),
  ),
);

4-3 使用主题

  • 使用主题
    • 通过Theme.of(context)函数来获取主题，方法是查找最近主题，如果找不到就会找整个应用的主题。
    • 以下是一个完整的示例代码，将应用的主题颜色定义为绿色，在界面中间加了一个带有背景色的文本。
      • 代码中的Key是每一个Widget的唯一身份标识，是在组件创建以及渲染时生成的。
      • 将Key作为参数传入Widget中，Widget会依据指定名字生成Key，Key是可选参数。
      • 受React启发且参考了虚拟Dom的Diff算法，若节点有Key进行比对，可最大限度重用已有的节点。
      • Key能够提高性能，因此每个Widget都会构建方法，也都会有一个Key参数可选，贯穿着整个框架。
    • Key派生出Localkey和Globalkey，用途如下。
      • Localkey：直接继承自Key，应用于拥有相同父Element组件进行比较的情况。
      • Globalkey：通过Globalkey找到持有该Globalkey的Widget、State和Element。
import 'package:flutter/material.dart';
import 'package:flutter/foundation.dart';

void main() {
  runApp(MyApp());
}

class MyApp extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    final appName = '自定义主题';

    return MaterialApp(
      title: appName,
      // 主题配置
      theme: ThemeData(
        // 应用程序整体主题的亮度
        brightness: Brightness.light,
        // App主要部分的背景色
        primaryColor: Colors.lightGreen[600],
        // 前景色(文本、按钮等)
        accentColor: Colors.orange[600]),
    );
  }
}

class MyHomePage extends StatelessWidget {
  // 标题
  final String title;
  // 接收title值Key为widget的唯一标识
  MyHomePage({required Key key, required this.title}) : super(key: key);

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(
        title: Text(title),
      ),
      body: Center(
        child: Container(
          // 获取主题的accentColor
          color: Theme.of(context).accentColor,
          child: Text(
            '带有背景色的文本组件',
            // 获取主题的文本样式
            style: Theme.of(context).textTheme.titleSmall,
          ),
        ),
      ),
      floatingActionButton: Theme(
        // 使用copyWith的方式获取accentColor
        data: Theme.of(context).copyWith(accentColor: Colors.grey),
        child: FloatingActionButton(
          onPressed: null,
          child: Icon(Icons.computer),
        ),
      ),
    );
  }
}
